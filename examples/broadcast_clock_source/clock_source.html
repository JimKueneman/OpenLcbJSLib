About this exerciser

Drives every WASM export tied to BroadcastTimeS Apr 25, 2021. Pick a clock identity, attach as producer (or consumer for testing), and use the panels below to send each event type defined in spec Section 4 and trigger the library-managed sequences from Sections 6.1, 6.3, 6.4 and 6.5.

Local Clock Advance
start(clockId) / stop(clockId) drive the library's internal time advance for this clock. While running, the library also emits the Section 6.2 Report Time stream and the Date Rollover sequence.

Producer Reports (wire)
Spec Section 4.1-4.4, 4.10-4.11. Each button forces an immediate emission of the corresponding Report event using the values staged below — bypassing any library-internal rate limiting.
Rate is stored as a 12-bit signed fixed-point per spec Section 4.4: the raw value is rate × 4. Effective rate: 1.00x
Section 6.3 burst: Producer Identified Valid for Stop/Start, Rate, Year, Date, Time, plus the next-minute Report Time. This is what a Query reply sends and what the generator emits at startup. triggerSyncDelay schedules that burst 3 seconds out, matching the post-Set-event behavior of Section 6.5.

Outbound Set Commands
Spec Section 4.5-4.9. These are what a remote throttle would send to control a fast clock. The exerciser ships them so you can verify the producer's Section 6.5 echo + 3-second burst behavior end-to-end on the bus, even with no second node present.
The Time/Date/Year/Rate Set commands re-use the values staged in the Reports panel above.
