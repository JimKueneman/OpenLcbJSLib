Connection

Connects one virtual OpenLCB node to a GridConnect WebSocket server (JMRI, an OpenLCB hub, etc.). Open the browser console to see the full protocol trace.
When checked, the node advertises Stream Protocol support in its PIP and Memory Configuration Options replies. Peers that prefer streams (none yet — JMRI still uses datagrams) may then choose streams over datagrams for memory reads/writes.

Send PC Event Report
Publish any event ID on the bus. Other nodes that consume this event will see it land via their onPcEventReport hook.

Query a peer
On connect, every node on the bus is asked to identify itself and we automatically request its SNIP. Pick one from the dropdown to fill its Node ID + alias, or type them in directly. All replies are also appended to the Log below.
