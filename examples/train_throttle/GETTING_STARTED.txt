GETTING STARTED
===============

1. Place openlcb_user_config.js where your application can import it.
   The default import path is:

       import { PSI } from '../../src/openlcb/constants.js';

   If you put the file in a different location, edit that path so it points
   to OpenLcbJSLib/src/openlcb/constants.js relative to this file.

2. Place cdi.xml (and fdi.xml for train nodes) alongside the .js file or
   wherever your application loads them from.  The byte arrays embedded in
   openlcb_user_config.js are the canonical wire-format representation;
   the XML files are the human-readable source.

3. In your application code, import and pass the parameters to createNode(),
   then call registerEvents() to register all the producers/consumers you
   selected in the Node Wizard:

       import { OpenLcb } from 'openlcb-js-lib';
       import { NODE_ID, OpenLcbUserConfig_node_parameters }
           from './openlcb_user_config.js';
       import { registerEvents } from './register_events.js';

       const lib = await OpenLcb.create({ /* transport options */ });
       const node = lib.createNode(NODE_ID, OpenLcbUserConfig_node_parameters);
       registerEvents(node);

NOTES
=====

  - mfgVersion (4) and userVersion (2) are SNIP string-count constants
    fixed by the OpenLCB spec.  Do not change them.

  - All address-space blocks are emitted with explicit "present" flags.
    The wrapper ignores blocks where present is false.

  - Buffer depths (max producers, consumers, nodes, etc.) are baked into
    the WASM at build time.  They are not tunable from this file.

  - To regenerate or modify these files, reload the project in Node Wizard
    using the Save/Load Project feature (use the <type>_project.json file).
